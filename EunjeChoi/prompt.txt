## 미션 개요 (Level 3)
- Arena: 5m × 3m, 최대 고도 1.25m
- 시퀀스: 이륙 패드 → 중간 구역 통과 → 착륙 패드 탐색 → 착륙 → 재이륙 → 복귀 → 이륙 패드 착륙
- 시간 제한: 3분 (초과 시 0점), 장애물 충돌 시 −2점/회
- 장애물: 13×13×125cm 박스(수직) + 13×13×125cm 바(수평 눕혀진 것), 위치/수량 미공개
- 착륙 패드: 30×30cm, 10cm 높이, 위치 미공개 (Landing Region 내 탐색)
- 이륙 패드 좌표는 2026-06-01 10시에 공개됨

## 좌표계
- EKF 리셋 후 이륙 패드가 원점 (0, 0, 0)
- +x: 착륙 구역 방향, +y: 왼쪽, +z: 위
- Arena 구역: START_REGION_X=1.5m, MIDDLE_REGION_X=2.0m, LANDING_REGION_X=1.5m

## 하드웨어 스펙
- Crazyflie 2.1 Brushless (보호 프레임 포함 15×15cm, 대각선 약 0.106m)
- Multi-ranger 덱: Front/Back/Left/Right/Up (수평), Down=range.zrange (수직 하향)
- 센서 값: mm 단위 uint16_t, 8000 이상이면 미감지(None 처리)
- Radio URI: radio://0/80/2M/E7E7E7E7E5

## 구현된 파일 (8개)

### config.py
- 모든 상수 및 파라미터 관리
- TAKEOFF_PAD_X / TAKEOFF_PAD_Y: 2026-06-01 공개 후 직접 입력
- 주요 값: FLIGHT_Z=0.5m, NAV_SPEED=0.3m/s, SCAN_SPEED=0.2m/s,
  STOP_THRESHOLD=0.25m, INFLATION_RADIUS≈0.156m, OCCUPANCY_GRID_RES=0.05m
- EKF 좌표 경계 헬퍼 함수 포함 (ekf_landing_region_start 등)

### shared_state.py
- threading.Lock 기반 thread-safe 데이터 버스
- 속성: current_state, pose(x,y,z,yaw), battery_pct, elapsed_time,
  emergency_flag, occupancy_grid(numpy), height_map_edges, pad_candidates
- trigger_emergency() 메서드: GUI 비상착륙 버튼에서 호출

### sensors.py
- SensorHub: 3개 LogConfig (ranges@100ms, pose@50ms, battery@1000ms)
- RangeData, SensorData 데이터클래스
- EdgeDetector: EMA 베이스라인 기반 z-ranger 신호에서 entry/exit 이벤트 감지
  (EDGE_BASELINE_ALPHA=0.02, EDGE_THRESHOLD=0.05m)
- EdgeEvent: kind('entry'/'exit'), x, y, timestamp
- edge_queue(Queue)로 미션 루프에 이벤트 전달

### mapping.py
- OccupancyGrid: numpy 그리드, Bresenham 레이캐스팅, 장애물 inflation
  - 셀 상태: FREE=0, UNKNOWN=1, OCCUPIED=2, INFLATED=3
  - update_all_rays(): yaw 보정해서 4방향 센서 모두 업데이트
  - snapshot(): thread-safe numpy 복사본 반환
- HeightMap: z-ranger edge 이벤트 쌍 매칭으로 착륙 패드 후보 추출
  - 같은 행(row)의 entry/exit 쌍 매칭 → 폭 20~45cm이면 패드 후보
  - 인접 후보 병합 후 PadCandidate(cx, cy) 리스트 반환

### navigator.py
- astar(): 그리드 스냅샷 위에서 8방향 연결 A* 탐색
- simplify_path(): LOS(Line-of-Sight) pruning으로 경유점 단순화
- FrontierNavigator: BFS로 +x 방향 가장 멀리 도달 가능한 FREE 셀 탐색
- LawnmowerNavigator: ㄹ자 패턴 waypoint 생성 (행 간격 SCAN_ROW_SPACING=0.25m)

### controller.py
- init_ekf(): kalman.resetEstimation 파라미터 리셋 + 2초 대기
- takeoff() / land(): HighLevelCommander 비차단 호출
- go_to_nonblocking(): 거리/속도로 duration 계산 후 HL go_to 호출, duration 반환
- send_hover(): send_hover_setpoint(vx_body, vy_body, yaw_rate, z) 래퍼
- stop_motion(): 제자리 호버링
- clamp_z(): 천장 근접 시 고도 낮춤

### gui.py
- matplotlib TkAgg 백엔드, figsize=(14,8), dark theme
- 구성: 상태바(상단) / 점유 격자 맵(좌) / 엣지 맵+패드 후보(우) / 비상착륙 버튼(하단)
- FuncAnimation 250ms 주기 갱신
- 점유 격자: FREE=흰색, UNKNOWN=회색, OCCUPIED=검정, INFLATED=어두운 회색
- 드론 위치(청록 점), 구역 경계선(노란 점선) 표시
- 비상착륙 버튼 클릭 → shared.trigger_emergency()

### mission.py (메인)
- EmergencyException: emergency_flag 감지 시 발생, 어디서든 착륙 처리
- _step(): 센서 읽기 + 맵 업데이트 + SharedState 동기화, 20Hz 루프의 기본 단위
- _navigate_to(): go_to_nonblocking 후 도착/장애물 감지 루프
  - 이동 방향 센서 < STOP_THRESHOLD → hl_commander.stop() 후 False 반환
- 상태 시퀀스 (순차 실행):
  TAKEOFF → ROTATION_SCAN → NAV_TO_LANDING → LANDING_REGION_SCAN
  → PAD_CONFIRM → LANDING_ON_PAD → TAKEOFF_FROM_PAD → TURN_AROUND
  → NAV_TO_START → LANDING_ON_START → DONE
- main(): TAKEOFF_PAD_X/Y 검증 → SyncCrazyflie 연결 → 미션 daemon 스레드 시작
  → GUI.start()로 메인 스레드 블록

## 각 상태 전이 조건
| 상태 | 전이 조건 |
|------|-----------|
| TAKEOFF | sleep(2.5) 경과 |
| ROTATION_SCAN | angle/rate 초 경과 (시간 기반) |
| NAV_TO_LANDING | EKF x ≥ ekf_landing_region_start() |
| LANDING_REGION_SCAN | 모든 ㄹ자 waypoint 완주 |
| PAD_CONFIRM | confirm_count ≥ needed 또는 전체 후보 시도 완료 |
| LANDING_ON_PAD | land() + sleep(2.8) |
| TAKEOFF_FROM_PAD | takeoff() + sleep(2.5) |
| TURN_AROUND | 180° 회전 완료 |
| NAV_TO_START | A* 경유점 모두 도달 |
| LANDING_ON_START | land() + sleep(2.8) → DONE |

## 아직 해결되지 않은 이슈 / 개선 여지
- TAKEOFF_PAD_X / TAKEOFF_PAD_Y를 2026-06-01에 입력해야 함
- matplotlib 설치 필요: pip install matplotlib
- 실제 비행 테스트 후 FLIGHT_Z, NAV_SPEED, EDGE_THRESHOLD 튜닝 필요
- 복귀 경로(A*) 재계획 로직이 단순함 — 재시도 1회 후 포기
- Landing pad 못 찾을 경우 fallback이 현재 위치에 착륙하는 단순 처리
